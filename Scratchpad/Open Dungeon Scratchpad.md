## Scaffolding
Things that are still required to establish the core:
1. Create a clear workflow for creating rulesets. (And we'll come back to this) 
	- Define file property metadata &  _how_ to connect rules together. Like when writing the rules, how do you define "this relies on". And how do we make it so one rule's definition can build on another? I.e. how can we define "if rule(set) ABC exists, this trait interacts with it by XYZ" 
2. Identify what we want as core rules. These are more or less immutable rules that are going to be required for any other rulesets to be created on top. Also as part of core, we need to have a "meta" ruleset. i.e. "the DM gets the final say in what a ruling is" 
	- We'll need to define at least a framework/scaffold of what we expect will be needed, even if the individual rules are not fleshed out. 
3. Define some core tenants that we'd want to adhere to
4. Define some failings of other systems that we want to improve upon or fix.
	- i.e. D&D action economy  
5. Define things that other systems do well, why they work, and how we can use those successes as inspiration.

## Hierarchy 
Using an atomic-like structure, ODe0.1 intends to allow for mixing and matching different rules 
The working hierarchy looks as follows: 
1. Trait
2. Feature/Rule
3. Module
4. Engine

## Core
The Core Module is an engine of how the rest of the game will be built. Some core rules need to exist to ensure other rules can be built up atop it.
